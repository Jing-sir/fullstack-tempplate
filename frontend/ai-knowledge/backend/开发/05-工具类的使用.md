# 工具类的使用（UPay）

## 1. 文档说明

本文基于 UPay 代码仓（重点 `upay-common`）梳理常用工具类的定位、典型用法与使用注意事项，便于在业务开发中统一调用方式。

---

## 2. 工具类总览（建议优先使用）

| 工具类 | 主要用途 | 典型场景 |
|---|---|---|
| `RedisLockUtil` | Redis 分布式锁、缓存回填 | 防重、并发互斥、缓存击穿保护 |
| `FileValidatorUtil` | 上传文件格式/大小校验 | KYC 图片、证件上传 |
| `ExcelUtil` | Excel 导出/读取 | 列表导出、批量导入读取 |
| `DateTimeUtil` | 时间格式化、时区转换、时间戳转换 | 报表时间、跨时区展示、参数转换 |
| `PasswordUtil` | 密码加密与校验（BCrypt） | 账号密码存储、登录校验 |

---

## 3. 详细使用

## 3.1 RedisLockUtil（推荐）

代码位置：

- `upay-common/src/main/java/com/upay/common/util/RedisLockUtil.java`

### 3.1.1 能力

- `lock/setIfAbsent`：带 TTL 的互斥锁
- `spinLock`：轻量自旋锁
- `tryGetAndUpdateCache`：缓存未命中时“加锁回填”

### 3.1.2 典型写法

```java
@Resource
private RedisTemplate<String, Object> redisTemplate;

public void doBusiness(String accountId) {
    RedisLockUtil lockUtil = new RedisLockUtil(redisTemplate);
    String lockKey = "biz:lock:" + accountId;
    boolean locked = lockUtil.lock(lockKey, "1", 30, TimeUnit.SECONDS);
    if (!locked) {
        throw new AppServerException(ResultCodeEnum.REQUEST_TOO_OFTEN);
    }
    try {
        // 业务逻辑
    } finally {
        lockUtil.del(lockKey);
    }
}
```

### 3.1.3 注意事项

- 锁 key 必须包含业务唯一标识（如 accountId / orderNo）
- 必须设置过期时间（TTL）
- 必须在 `finally` 中释放锁

## 3.2 FileValidatorUtil

代码位置：

- `upay-common/src/main/java/com/upay/common/util/FileValidatorUtil.java`

### 3.2.1 能力

- 对上传文件进行后缀与大小校验
- 当前实现重点覆盖 KYC 证件图片：`jpg/jpeg/png` 且 `< 4MB`

### 3.2.2 典型写法

```java
@PostMapping("/upload")
public JsonResult<String> upload(@RequestParam("file") MultipartFile file) {
    FileValidatorUtil.isValidFileFormat(file, FileTypeEnum.CERTIFICATE);
    // 校验通过后再走 OSS/S3 上传
    return new JsonResult<>("ok");
}
```

### 3.2.3 注意事项

- 先校验、后上传
- 前端限制与后端校验保持一致（格式/大小）
- 校验失败会抛业务异常（如 `KYC_IMAGE_ERROR`）

---

## 3.3 ExcelUtil

代码位置：

- `upay-common/src/main/java/com/upay/common/util/excel/ExcelUtil.java`

### 3.3.1 能力

- `export(...)`：导出 Excel（支持 headerClass / headers map）
- `readRowAsObject(...)` / `readRowsAsObject(...)`：读取 Excel 数据
- 导出最大数量限制：`MAX_NUM = 20000`

### 3.3.2 典型写法（导出）

```java
@PostMapping("/export")
public void export(HttpServletResponse response, @RequestBody QueryReq req) {
    ExcelUtil.export(response, "客户列表", CustomerExportRes.class, () -> {
        // 查询导出数据
        return customerService.queryExportList(req);
    });
}
```

### 3.3.3 注意事项

- 列头为空会抛异常
- 导出数量超过上限会被截断到 20000
- 导出接口建议单独配置权限码

---

## 3.4 DateTimeUtil

代码位置：

- `upay-common/src/main/java/com/upay/common/util/DateTimeUtil.java`

### 3.4.1 能力（常用）

- 时间与字符串互转
- 时间戳与日期互转
- UTC+0 / UTC+4 / UTC+8 转换
- 日期区间与偏移计算

### 3.4.2 典型写法

```java
String begin = DateTimeUtil.timestampToDate8("1719811200000");
String utc0 = DateTimeUtil.convertParamToUtcTime("2026-04-03 12:00:00", "+UTC+8");
Long ts = DateTimeUtil.dateToEpochMilli("2026-04-03 12:00:00");
```

### 3.4.3 注意事项

- 优先使用明确时区的方法，避免“默认时区”歧义
- 入参为空时部分方法返回 `null`，调用侧注意判空

---

## 3.5 PasswordUtil

代码位置：

- `upay-common/src/main/java/com/upay/common/util/PasswordUtil.java`

### 3.5.1 能力

- `encrypt`：BCrypt 加密
- `verify`：明文与密文校验

### 3.5.2 典型写法

```java
String encrypted = PasswordUtil.encrypt(rawPassword);
boolean ok = PasswordUtil.verify(loginPassword, encryptedPasswordFromDb);
if (!ok) {
    throw new AppServerException(ResultCodeEnum.PASSWORD_ERROR);
}
```

### 3.5.3 注意事项

- 密码只存密文，不可逆
- 校验必须用 `verify`，不要自行字符串比对

---

## 4. 使用建议（团队统一）

1. **优先复用 `upay-common` 工具类**，避免业务模块重复造轮子。  
2. **工具类负责通用能力，业务规则留在 service**，避免工具类业务化。  
3. **异常统一抛业务异常枚举**，不要在业务中硬编码错误码。  
4. **涉及资金/审核/导出的接口**，工具类调用后仍需做权限与审计日志。

## 5. 快速清单

- 分布式锁：`RedisLockUtil.lock(...)/del(...)`
- 上传校验：`FileValidatorUtil.isValidFileFormat(...)`
- 导出 Excel：`ExcelUtil.export(...)`
- 时间转换：`DateTimeUtil.*`
- 密码处理：`PasswordUtil.encrypt/verify`

以上是当前 UPay 中最常用、最建议统一使用的工具类入口。
