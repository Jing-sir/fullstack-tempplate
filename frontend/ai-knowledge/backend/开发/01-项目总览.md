# UPay 项目总览

## 1. 项目定位

`UPay` 是一个以支付业务为核心的多模块 Java 后端工程，采用 Maven 聚合管理，按“公共能力 + 业务域服务 + API 接入层 + SDK”组织代码。

从目录结构看，工程同时覆盖：
- 管理端/开放平台/API 接入
- 用户、钱包、卡、渠道、借贷、收益、换汇等业务域
- 通知、任务、迁移、聚合等支撑域

## 2. 规模与结构（基于目录与 POM）

- 根聚合模块数：`26`（来自根 `pom.xml`）
- 启动类数量：`24`（`*Application.java`）
- 含 `docker/` 的模块：`20`

## 3. 顶层模块分组

### 3.1 公共与基础能力
- `upay-model`：领域模型与基础数据对象
- `upay-common`：公共组件与通用能力
- `upay-asset`：资产相关共享能力

### 3.2 API 接入层（可独立启动）
- `upay-sys-api`
- `upay-open-api`
- `upay-external-api`
- `upay-app-api`
- `upay-broker-api`
- `upay-caas-api`

### 3.3 业务域服务层（多数为 client/service 拆分）
- 用户：`upay-user-server`
- 钱包：`upay-wallet-server`
- 卡相关：`upay-card-server`、`upay-card-extend-server`
- 借贷：`upay-borrow-server`
- 券商：`upay-broker-server`
- 通道：`upay-channel-passto`、`upay-ditch-server`
- 收益/换汇：`upay-earn-server`、`upay-swap-server`
- 收单：`upay-acquirer-server`
- 迁移：`upay-migrate-server`
- 通知域：`upay-notice-server`
- 聚合域：`upay-sys-aggregation-server`

### 3.4 支撑模块
- `upay-task`：定时任务与批处理
- `upay-notify`：通知发送与编排
- `upay-open-sdk`：独立 Java SDK（`jar`，版本 `1.0.8`）

## 4. 技术栈画像

从各模块 `pom.xml` 可见主要技术栈：
- Java 版本：`JDK 8`（`source/target 1.8`）
- Spring：`Spring Boot 2.x`（多数模块继承 `spring-boot-starter-parent`）
- 服务治理：`Dubbo 2.7.3` + `Nacos`（注册/配置）
- 数据访问：`MyBatis` / `MyBatis-Plus`
- 数据库与缓存：`MySQL` + `Redis(Jedis)`
- 消息：`spring-boot-starter-amqp`（RabbitMQ）
- 连接池：`Druid`
- API 文档：`Swagger/Springfox` + `swagger-bootstrap-ui`
- 常见增强：`PageHelper`、`Lock4j`

## 5. 代码与资源组织特征

### 5.1 Java 包结构（典型）

以 `upay-sys-api` / `upay-open-api` / `upay-app-api` 为例，普遍包含：
- `common`（auth/config/filter/interceptor/monitor/util）
- `controller`
- `service` + `service/impl`
- `mapper`
- `model`（`dto` / `req` / `res`）
- 部分模块含 `task`、`strategy`、`handler`

### 5.2 配置与资源规范（高度统一）

多数可运行模块的 `src/main/resources` 下均具备：
- `bootstrap.yml`
- `bootstrap-dev.yml`
- `bootstrap-test.yml`
- `bootstrap-pre.yml`
- `bootstrap-prod.yml`
- `logback-spring.xml`

并按模块需要提供 `mapper/*.xml`（数量差异较大，说明业务复杂度不同）。

## 7. 架构解读（建议作为沟通口径）

- 这是“**聚合仓 + 领域拆分 + 接口契约前置**”架构。
- `client` 模块承载跨服务调用契约，`service` 模块承载业务实现。
- API 层（sys/open/external/app/broker/caas）负责不同渠道入口与编排。
- 通知、任务、迁移、聚合模块作为横向支撑能力，服务于主业务域。
- `upay-open-sdk` 独立发布，面向外部集成方。

## 8. 快速上手建议（面向新同学）

1. 先阅读根 `pom.xml` 的 26 个模块与依赖关系。
2. 按“入口层 -> 域服务 -> 公共模块”顺序看代码：
    - `upay-sys-api` / `upay-open-api`
    - `upay-user-server` / `upay-wallet-server`
    - `upay-common` / `upay-model`
3. 重点关注每个服务的 `bootstrap-*.yml` 与 `logback-spring.xml`。
4. 联调时优先确认 Nacos、Dubbo、MySQL、Redis、MQ 的连通性。
