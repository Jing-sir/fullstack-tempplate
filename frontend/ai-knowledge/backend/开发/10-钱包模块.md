# 钱包模块

## 1. 模块定位

`UPay 的核心钱包业务域，负责充币、提币、地址管理、资产快照、AML 风控、Tron 能量管理、快捷充值等全链路钱包能力。

**双子模块拆分：**

| 子模块 | 职责 |
|---|---|
| `upay-wallet-client` | 对外暴露 Dubbo 接口契约（`I*Client` 接口 + 请求/响应 Model）|
| `upay-wallet-service` | 业务实现、Mapper、第三方对接、定时任务入口 |

---

## 2. 整体架构

```
上游调用方（upay-sys-api / upay-app-api / upay-task 等）
        │  Dubbo RPC
        ▼
┌─────────────────────────────────────┐
│         upay-wallet-client          │  ← 接口契约层
│  ITransferClient / IAddressClient   │
│  IAmlClient / ICoinClient / ...     │
└────────────────┬────────────────────┘
                 │ Dubbo @Service 实现
┌────────────────▼────────────────────┐
│        upay-wallet-service          │  ← 业务实现层
│  impl/          ← Dubbo 接口实现     │
│  service/impl/  ← 内部 Spring Service│
│  mapper/        ← MyBatis-Plus      │
│  common/        ← 第三方对接         │
│    ├── fotpay/  ← FortPay Retrofit  │
│    └── safeheron/ ← SafeHeron SDK   │
└────────────────┬────────────────────┘
                 │
        ┌────────┴────────┐
        ▼                 ▼
   三方钱包           upay-asset
Safeheron/FortPay/UE  (余额增减)
```

**三套钱包体系对比：**

| 钱包 | 类型 | 地址创建 | 充值感知 | 提币发起 |
|---|---|---|---|---|
| **Safeheron** | MPC 托管（主流）| API 批量创建 | Webhook 回调 | `createTransactionsV3` |
| **FortPay** | 第三方托管 | API 创建 | Webhook 回调 | FortPay 提现 API |
| **UE 钱包** | UEEX 交易所内部 | HTTP 接口 | 定时轮询 | 不独立发起 |

详细对接文档见：
- [04-Safeheron钱包.md](11-Safeheron钱包)
- [04-UE钱包.md](12-UE钱包)
- [06-FortPay钱包.md](13-FortPay钱包)

## 3. 充值核心流程

```
用户充币（链上转账）
    │
    ▼
三方钱包 Webhook 回调 → HTTP 接收（upay-external-api）
    │  Dubbo 调用
    ▼
IDepositWithdrawClient.deposit(DepositWithdrawNotifyReq, Coin, Chain)
    │
    ├─ 防重校验（txid + toAddress 唯一）
    ├─ 按 WalletTypeEnum 分支：
    │     Safeheron → depositSafeReceived()
    │     FortPay   → checkDeposit()
    │     UE        → depositUeReceived()
    │
    ├─ AML 风控检测
    │     通过 → 继续入账
    │     拦截 → riskNotice 通知，Transfer.riskCoinStatus = FROZEN
    │
    ├─ 资产入账：IAssetBalanceService.addBalance(accountId, coinId, amount)
    ├─ 更新 Transfer.state = SUCCESS
    ├─ 发消息通知：sendDepositToAccountMsg（站内信 + 邮件）
    └─ 账单记录：sendMerchantBillSaving
```

**Transfer 充值状态流转：**

```
CONFIRMING（确认中）→ SUCCESS（成功）
                   → FROZEN（AML 冻结）
                   → FAILED（失败）
```

## 4. 提币核心流程

```
用户发起提币
    │
    ▼
WithdrawService.checkTransfer(TransferCheckReq)
    ├─ 金额校验（最小提币额、单笔限额）
    ├─ 地址合法性校验（链格式、AML 黑名单）
    ├─ 交易密码验证
    └─ 提币频次限制检查
    │
    ▼
冻结余额：IAssetBalanceService.freeze(accountId, coinId, amount)
    │
    ▼
创建 Transfer 记录（state=PROCESSING）
    │
    ├─ WalletTypeEnum = SAFE_HERON
    │     → ISafeHeronTransactionsServiceV2.withdraw(OrderWithdrawDto)
    │       → Safeheron createTransactionsV3 API
    │
    └─ WalletTypeEnum = FORT_PAY
          → IFortPayService.withdraw(...)
            → FortPay 提现 API
    │
    ▼
等待三方回调通知结果
    │
    ├─ 成功：扣减冻结余额，Transfer.state = SUCCESS
    │         sendWithdrawSuccessMsg
    └─ 失败：解冻余额，Transfer.state = FAILED
```

**Transfer 提币状态流转：**

```
PROCESSING（处理中）→ SUCCESS（成功）
                   → FAILED（失败，自动解冻）
                   → CHECK（待审核，大额提币）
```

## 5. 对外 Dubbo 接口总览

### 5.1 核心接口

| 接口 | 职责 | 关键方法 |
|---|---|---|
| `ITransferClient` | 充提币订单管理 | `transferList`、`checkTransfer`、`depositList`、`amlReceipt`、`afreshTransfer` |
| `IDepositWithdrawClient` | 充提币回调处理 | `deposit`、`depositSafeReceived`、`depositUeReceived`、`withdrawUeReceived` |
| `IAddressClient` | 地址管理 | `createSafeHeronAddress`、`createUeAddress`、`createFortPayAddress`、`getAddress` |
| `ICoinClient` | 币种 & 链信息 | `coinList`、`chainListSel`、`coinListSel`、`checkCoinEnable`、`synchCoinInfo` |
| `IAmlClient` | AML 风控 | `uploadAmlMaterial`、`depositRefund`、`riskNotice`、`getPageAmlList`、`handleAmlRefund` |
| `IAssetClient` | 资产快照 & 同步 | `walletAssetBalanceSnapshot`、`userAssetSnapshot`、`walletAssetBalanceSync` |
| `IFortPayClient` | FortPay 对接 | `checkDeposit`、`checkWithdraw`、`depositWithdrawNotify`、`riskScore` |
| `IReplenishFeeClient` | Tron 能量租用 | `rentingEnergy`、`executionReplenishFee`、`confirmRentingTrx` |
| `IWalletQuickDepositClient` | 快捷充值 | `check`、`create`、`findInfo` |

### 5.2 补充接口

## 6. 关键请求/响应模型

**提币请求 `WithdrawReq`：**

```java
String itemId;    // 币种ID
String amount;    // 提币数量
String tradePwd;  // 交易密码
String address;   // 目标地址
String chainTag;  // 链标识（如 TRC20、ERC20）
String remark;    // 备注（可选）
```

**充提通知请求 `DepositWithdrawNotifyReq`（三方钱包回调体）：**

```java
WalletTypeEnum walletTypeEnum; // 钱包来源：UE / FORT_PAY / SAFE_HERON
String exchange_id;            // 三方订单ID
String status;                 // success / faild
String item_id;                // 币种ID
String chain_tag;              // 链标识
String number;                 // 数量
Integer address_type;          // 1=内部地址 2=外部地址
Integer order_type;            // 1=充币 2=提币
String txid;                   // 链上交易 Hash
String from_address;
String to_address;
String user_id;                // 平台 UID
String sign;                   // 签名
```
