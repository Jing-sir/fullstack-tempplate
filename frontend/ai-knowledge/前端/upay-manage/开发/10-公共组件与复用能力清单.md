# 10 公共组件与复用能力清单

## 用途

这个项目里很多需求表面看是“新增一个小功能”，实际上已经有现成组件或 hook 可以直接复用。

新增功能前先查这里，可以少造轮子。

## 常用选择器组件

### `src/components/SAgentChoose.vue`

用途：

- 代理商下拉

特点：

- 内部调用 `api.fetchAgentChoose()`
- 默认支持“全部”
- 会抛出 `onChange` 和 `setAgentName`

适合场景：

- 代理商筛选
- 代理商关联配置

### `src/components/SCoin.vue`

用途：

- 币种下拉

特点：

- 内部调用 `api.fetchCoinChoose()`
- 可联动 `agentId`、`walletSiteId`
- 支持默认选中、返回完整币种对象

适合场景：

- 币种筛选
- 钱包 / 链路联动

### `src/components/SChain.vue`

用途：

- 链下拉

特点：

- 依赖 `coinId`
- 内部调用 `api.fetchChainChoose()`
- 可联动 `agentId`、`walletSiteId`

适合场景：

- 币种选完后继续选择链

### `src/components/SUserID.vue`

用途：

- 用户 UID 下拉

特点：

- 内部调 `api.fetchSysUserListChoose()`
- 默认一次拉固定分页大小的数据

适合场景：

- 用户维度筛选
- UID 指定查询

## 常用 hooks / composables

### `src/use/useTableConf.ts`

用途：

- 手写页统一分页、搜索、重置

适合场景：

- `script setup` 或手写列表页

注意：

- 自带 `PagingDefaultConf`
- 自带节流搜索

### `src/use/useOnActivated.ts`

用途：

- 统一处理带 `#no-refresh` 的激活刷新逻辑

适合场景：

- keep-alive 页面
- 从详情返回列表页

### `src/use/useModalHandler.ts`

用途：

- 统一 modal 的开关和确认 / 取消流

适合场景：

- 新增本地弹窗页时，不想重复手写 `open`、`onCancel`、`onOk`

### `src/use/useUpload.ts`

用途：

- 图片上传前校验与压缩

特点：

- 校验 jpg / jpeg / png
- 限制大小
- 用 `compressorjs` 压缩

### `src/use/useMultipleMenus.ts`

用途：

- 跨菜单权限判断

适合场景：

- 当前页面要展示一个跳往其他菜单域页面的入口

### `src/use/useButtonRole.ts`

用途：

- 辅助获取按钮权限或按钮菜单项

适合场景：

- 需要进一步按当前菜单树拆按钮信息

### `src/use/useTabsRole.ts`

用途：

- 基于 `roleMenu` 过滤有权限的 tabs，并判断 tab 内按钮是否可见

特点：

- `fetchShowTabs`：过滤只展示有权限的 tab 列表
- `isShowTabsBtn(btnRole)`：判断 tab 内某个按钮是否可见
- 自动将第一个有权限的 tab 设为激活项

适合场景：

- 页面有多 tab，且每个 tab 需要独立权限控制

### `src/use/useFetchTableData.ts`

用途：

- 手写页的通用列表数据拉取封装（非 config-overall 场景）

特点：

- 内置分页 `pageNo / pageSize / pageTotal`
- 内置 `loading` + NProgress
- `runAsync(params)` 手动触发，`resetAndLoad(params)` 重置分页后触发
- `filterColumns(keys?)` 支持动态显隐列
- 默认 `onBeforeMount` 自动加载，`manual: true` 时不自动加载

适合场景：

- `script setup` 手写列表页，不想用 config-overall 但需要统一分页逻辑

### `src/use/useFormHandler.ts`

用途：

- 统一 form 表单的状态、重置、校验逻辑

特点：

- `formState`：响应式表单状态
- `resetFields()`：重置为 defaultState 深拷贝
- `validateFields()`：触发全量校验，返回 Promise
- `setFields(fields)`：合并更新部分字段

适合场景：

- 新增 / 编辑弹窗的 form 状态管理

### `src/use/useValidatorConf.ts`

用途：

- 提供常用 Ant Design Vue 表单验证器

包含验证器：

- `validateEmail` / `validatePass` / `validateCode` / `validateFounds`
- `validateNum`（正整数）/ `validateEn`（纯英文）
- `validateEnStr`（英文+空格+特殊字符）/ `validateNotCn`（非中文）
- `validatorTwoNum` / `validatorMaxNum`（最多两位小数）
- `validateAddressName` / `validatorAddressLine1` / `validatorPostCode`（地址相关）

适合场景：

- 表单 `:rules` 配置中复用，不需要重复手写正则

### `src/use/useAgentConf.ts`

用途：

- 代理商相关下拉选项的统一获取

包含：

- `agents` / `getAgents(code)`：代理商名称列表
- `agentTypes` / `getAgentTypes()`：代理类型
- `agentCustomers` / `getAgentCustomers(groupIds)`：代理商客户
- `agentCustomerGroup` / `getAgentCustomerGroup(agentId)`：代理商客户组
- `status` / `getStatus()`：转账申请状态

适合场景：

- 代理商相关筛选项，替代分散的 API 手动调用

### `src/use/useDitchOptions.ts`

用途：

- 渠道相关下拉选项统一获取

包含：

- `ditchArr` / `getDitchArr()`：渠道下拉
- `cardArr` / `getCardArr(ditchId)`：渠道卡片下拉
- `feeTypes` / `getFeeTypes()`：费用类型
- `reductionTypes` / `getReductionTypes()`：减免类型

适合场景：

- 渠道管理相关筛选项

### `src/use/useVaCommon.ts`

用途：

- VA 模块公用下拉数据获取

包含：

- `vaCoinArr` / `fetchVaCoinArr()`：VA 币种列表
- `vaTypeArr` / `fetchVaTypeArr()`：冻结类型列表
- `vaChannelArr` / `fetchVaChannelArr()`：渠道类型列表

适合场景：

- VA 账户相关页面的筛选项

### `src/use/useCountry.ts`

用途：

- 国家 / 省份 / 城市三级联动数据获取，支持模糊搜索

包含：

- `getCountryArr(val?)` / `countryArr`：国家列表（支持区号模式）
- `getProvinceArr(pid)` / `provinceArr`：省/州列表
- `getCityArr(pid)` / `cityArr`：城市列表
- `filteredOptions`：中英文 + 拼音 + 区号模糊过滤后的国家列表
- `filterOptions(val)`：防抖搜索触发

适合场景：

- 地址表单的国家/省/市三级联动

### `src/hooks/tableScroll.ts`

用途：

- 表格滚动到底部时追加下一页数据（虚拟滚动分批加载）

适合场景：

- 不需要分页组件、用滚动加载代替翻页的老页面

## 全局指令

### `v-has-btm`（`src/directives/hasBtm.ts`）

用途：

- 基于 `roleMenu` 判断权限，无权限时直接从 DOM 移除元素

用法：

```html
<a-button v-has-btm="'permissionKey'">操作</a-button>
```

注意：

- 这是指令形式的权限控制，与 `buttonPermissions()` 效果类似，但不需要写 `v-if`
- 适合只需要隐藏元素、不需要逻辑判断的简单场景

### `v-defempty`（`src/directives/whenEmpty.ts`）

用途：

- 值为 `null` / `undefined` 时自动显示 `--`

用法：

```html
<span v-defempty="value" />
```

### `v-only-number`（`src/directives/onlyNumber.ts`）

用途：

- 限制输入框只能输入数字，支持配置小数位数

用法：

```html
<a-input v-model:value="val" v-only-number:2 />
```

参数为允许的小数位数，`:0` 表示整数。

## 全局过滤器（`$filters`）

项目通过 `app.config.globalProperties.$filters` 注册了以下工具函数，模板里直接用 `$filters.xxx()` 调用，JS 中直接 import 对应文件使用。

### 时间

- `$filters.timeStampToDate(ts, format?)` → `src/filters/dateFormat.ts`
  - 时间戳 / 日期字符串转格式化字符串，默认 `YYYY-MM-DD HH:mm:ss`
  - 传空值返回 `''`

### 数字

- `$filters.toFixed(amount, fix, mode?)` → `src/filters/numberOperation.ts`
  - 截断小数位，默认 `fixed` 模式（银行家截断，不四舍五入）
  - `reg` 模式返回字符串，可避免浮点精度问题，超大数字场景用这个
- `$filters.toNonExponential(num)` → `src/filters/numberOperation.ts`
  - 科学计数法转普通数字字符串

### 金额格式化

- `$filters.dataThousands(value)` → `src/filters/dataThousands.ts`
  - 金额加千分符，如 `1234567.89` → `1,234,567.89`

### 字符串脱敏

- `$filters.strIntercept(str, length?)` → `src/filters/stringOperation.ts`
  - 保留前后 `length`（默认 3）位，中间替换为 `****`
  - 适合卡号、手机号等敏感信息显示

### Tab 按钮权限

- `$filters.isShowTabsBtn(btnRole)` → `src/filters/tabsBtnRole.ts`
  - 判断 `roleMenu` 里是否有该权限键，适合模板内 tab 内按钮显隐

### 数组排序

- `$filters.arraySort(arr, attr, rev?)` → `src/filters/arraySort.ts`
  - 按对象属性排序（支持 `.` 嵌套路径），`rev = -1` 倒序

## 工具函数（`src/utils/`）

常用的独立工具函数，直接 import 使用，不需要注册。

### 文件下载

```ts
import { downloadExcel } from '@/utils/common';
// 导出接口返回 blob 后用此函数触发下载，自动加时间戳后缀
downloadExcel(blobData, '文件名');
```

也有 `downloadUrl(url, name)` 用于直接下载 URL 链接。

### 复制到剪贴板

```ts
import { copyText } from '@/utils/common';
copyText('内容', true); // 第二个参数为 true 时自动提示"复制成功"
```

`src/utils/copyToClipboard.ts` 是另一个不弹 toast 的纯复制实现，两者二选一。

### 日期范围格式化

```ts
import { formatDateRange, formatDateRangeByNum } from '@/utils/common';
// formatDateRange：范围转字符串，第二个参数为 false 时不补 endOf('day')
// formatDateRangeByNum：范围转毫秒时间戳
```

### 脱敏显示

```ts
import { hideMiddleText } from '@/utils/common';
hideMiddleText('1234567890ab'); // 保留前后6位，中间显示 ...
```

### 精度计算

```ts
import exprCalc from '@/utils/math';
exprCalc('0.1 + 0.2'); // 基于 mathjs BigNumber，避免浮点精度问题
```

### XSS 过滤

```ts
import { filterXss } from '@/utils/common';
// 富文本内容展示前过滤，内置白名单
```

### `src/utils/constant.ts` 常量

- `PagingDefaultConf`：手写页分页默认配置（`pageSize: 20`，带 showTotal）
- 大量常用正则：`EMAIL` / `PASSWORD` / `INTER_NUMBER` / `IP_V4` / `IP_V6` / `IS_NOT_CN` 等
- `MaxTwoDecimals(max)` 工厂函数：动态生成限制小数位数的正则

## 开发时的复用顺序

新增功能时建议按这个顺序找复用能力：

1. 当前页面同目录兄弟组件
2. `src/components/*` 下的通用选择器 / 展示组件
3. `src/use/*` 下的复用 hook
4. `src/config-overall/*` 公共能力

## 使用建议

- 先看组件 props / emits，再决定是否直接复用
- 如果只差很小一点，不要复制出一个新组件，优先补最小兼容
- 如果组件已经是老风格实现，不要为了复用顺手重写
