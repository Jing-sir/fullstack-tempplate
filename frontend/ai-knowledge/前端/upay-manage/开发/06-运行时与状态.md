# 06 运行时与状态

## main.ts 负责注册什么

`src/main.ts` 里会注册这些关键能力：

- router
- pinia
- i18n
- `menuMixin`
- `treeMenu`
- `hasBtm` 指令
- `vue-request` 全局分页配置

还会挂一些全局能力：

- `$is2FA()`
- `buttonPermissions()` 来自全局 mixin
- `$getRoleMenu()`、`$buildArrTree()`、`$getChildren()` 来自 `treeMenu`

## Main.vue 是后台主壳

`src/Main.vue` 主要做这些事：

- 渲染整体后台布局
- 加载左侧菜单、头部和 tagsView
- 切路由时触发 `fetchBreadcrumb()`
- 拉取当前账号的菜单权限
- 控制 `router-view` 的 keep-alive

一个很关键的细节是：

- 当 `store.roleMenu.length` 为空时，不会进入正常业务页面渲染
- 页面会先显示加载态、网络异常态或错误态

所以“页面打不开”不一定是页面本身问题，也可能是菜单加载链路没成功。

## sideBar store

`src/store/sideBar.ts` 是权限和菜单的核心 store，状态包括：

- `isSidebar`
- `routes`
- `roleMenu`

关键方法：

- `fetchSidebarRoutes()`
  - 拉后端菜单并过滤异步路由
- `clearRoutes()`
  - 清空菜单和权限
- `updateIsSidebar()`
  - 控制左侧折叠

## user store

`src/store/user.ts` 主要负责：

- 当前用户信息 `userInfo`
- 标签列表 `tagList`
- `fetchUserInfo()`
- `fetchTagList()`

其中 `fetchUserInfo()` 还会把用户信息同步给加密 store。

## 全局菜单树工具

`src/hooks/treeMenu.ts` 注册了几个全局方法：

- `$getRoleMenu()`
- `$buildArrTree()`
- `$getChildren()`
- `$treeConvertArray()`

这些能力主要用于：

- 权限树构建
- 菜单树递归查找
- tabs / 子按钮权限判断

## 全局权限 mixin

`src/mixins/menuMixin.ts` 给所有页面提供了这些能力：

- `buttonPermissions(btn)`
- `getButtonsForMenu()`
- `isRoleBtn()`
- `permissionShow`

这也是为什么老页面和新页面里大量直接调用 `buttonPermissions('xxx')`。

## keep-alive 行为

`Main.vue` 里会根据 `tagsView` store 的 `visitedViews` 生成 keep-alive 列表。

这意味着：

- 某些页面在切走后不会完全销毁
- 页面里经常会配合 `onActivated` 做刷新或局部跳过刷新

如果你改的是列表页、详情页、tabs 页，这个行为需要一起考虑。

## 常见问题定位

### 页面一直在加载

优先检查：

- `fetchSidebarRoutes()` 是否成功
- 菜单接口是否报错
- `roleMenu` 是否为空

### 页面突然跳回登录页

优先检查：

- 接口返回码是否命中 `10005`、`10021`、`10024`
- token 是否已经过期

### 按钮权限时有时无

优先检查：

- 当前页面 `route.name`
- 当前菜单节点下是否真的有对应按钮权限
- 是否在 keep-alive / `onActivated` 逻辑里导致状态没刷新

### tabs 或局部权限区域不显示

优先检查：

- `treeMenu` 的树查找链路是否命中
- `buttonPermissions()` 或 `isRoleBtn()` 传入的权限键是否正确

## 其他 Store

除 `sideBar` 和 `user` 外，还有以下 store：

- `src/store/tagsView.ts`：标签页（浏览历史）管理，keep-alive 列表来源
- `src/store/encryption.ts`：存储加密 IV，供 `src/hooks/useEncryptData.ts` 使用
- `src/store/timezone.ts`：时区设置
- `src/store/theme.ts`：主题配置
- `src/store/config-overall/`：config-overall 页面的公共状态
- `src/store/country.ts` / `currency.ts` / `snapshot.ts` / `cancellation.ts` / `cardInventory.ts`：各业务域专用 store

## 全局指令

`src/main.ts` 注册了以下全局指令：

- `v-has-btm`（`src/directives/hasBtm.ts`）
  - 基于 `roleMenu` 判断权限，无权限时直接从 DOM 移除元素
  - 是 `v-if="buttonPermissions('xxx')"` 的指令替代形式
- `v-defempty`（`src/directives/whenEmpty.ts`）
  - 值为 `null/undefined` 时自动显示 `--`
- `v-only-number`（`src/directives/onlyNumber.ts`）
  - 限制输入框只能输入数字，`:参数` 指定小数位数，如 `v-only-number:2`

## 加密工具

`src/hooks/useEncryptData.ts` 提供 `encryptKey(key)` 和 `decrypt(data)` 方法，基于 AES-GCM，IV 动态从后端拉取并缓存到 `encryption` store。

不要在业务代码里自行写加密，有需要复用这里。

## 网络状态监听

`src/mixins/networkMixin.ts` 提供：

- Options API 形式：`networkMixin`（全局 mixin）
- Composition API 形式：`useNetwork()`

页面里通过 `this.networkState` 或解构 `networkState` 拿到当前网络状态（`ONE_LINE` / `OFF_LINE`）。

## 离开页面拦截

`src/mixins/beforeunloadMixin.ts` 提供：

- Options API 形式：`beforeunloadMixin`，通过 `isAllowClose` 控制是否弹出离开确认
- Composition API 形式：`useBeforeunload()`

适合有未保存内容时拦截用户关闭/刷新浏览器。

## 开发时的建议

- 遇到权限问题时，不要只盯页面文件
- 同时看 `Main.vue`、`router-setup.ts`、`sideBar.ts`、`menuMixin.ts`
- 遇到缓存或页面不刷新问题，优先排查 keep-alive 与 `onActivated`
