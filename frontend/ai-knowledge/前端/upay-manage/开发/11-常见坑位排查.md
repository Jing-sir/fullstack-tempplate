# 11 常见坑位排查

## 1. 路由文件存在，但页面还是进不去

优先检查：

- `src/routes/asyncRoutes.ts` 有没有注册该业务模块
- 当前页面 `meta.role` 是否在后端菜单 `component` 列表里
- 当前页面 `requiresAuth` 是否为 `true`

排查思路：

- 先看 `asyncRoutes.ts` 的 import 列表是否真的包含这个模块
- 再看模块里的 `meta.role` 是否和后端返回的 `component` 对齐

## 2. 页面能打开，但按钮不显示

先判断是哪类权限：

- 当前页面自己的按钮权限
  - 用 `buttonPermissions()`
- 其他菜单域下的权限
  - 用 `isMoreRoleBtn(parentKey, btnKey)`

如果这两类混用了，按钮很容易误判。

## 3. 跨菜单跳转按钮不显示

优先检查：

- `parentKey` 是否真的是目标父菜单的 `component`
- `btnKey` 是否真的是目标子页面或按钮权限的 `component`
- 当前账号是否真的拥有该父级菜单域权限

## 4. 页面返回列表后自动刷新 / 不刷新行为不一致

优先检查：

- 页面是否在 `onActivated` 里处理刷新
- 是否用了 `route.hash === '#no-refresh'`
- 是否可以直接复用 `src/use/useOnActivated.ts`

## 5. `config-overall` 页列表不出数据

优先检查接口返回格式是否包含：

- `list`
- `totalSize`
- `pageNo`
- `pageSize`

如果格式不对，优先在 API 层适配。

## 6. 导出接口请求成功，但文件没下来

优先检查：

- API 是否用了 `responseType: 'blob'`
- 页面是否用现有下载方法处理
- `config-overall` 的 `exportBtns` 是否补齐了 `handExport` 和 `exportFileTitle`

## 7. GET 请求传数组后参数丢失

`src/servers/index.ts` 里的 `get()` 会过滤数组类型参数。

所以：

- 如果某个 GET 接口必须带数组 query
- 不要默认照普通 `get()` 方式直接传

## 8. 老页面增量改动时样式越改越乱

当前项目规则是：

- 新增样式优先 Tailwind
- 老页面已有 SCSS 结构时，只做最小样式补丁
- 不要一边增量需求，一边扩写新的大段 SCSS

## 9. 类型检查一跑就是一堆报错

当前仓库没有干净的全量 TypeScript 校验基线。

所以更实用的做法通常是：

- 先看局部 diff 是否合理
- 再做目标文件的运行链检查
- 必要时做局部命令验证，而不是把全仓 `tsc` 通过当成唯一标准

## 10. 路由与页面目录命名不完全一致

不要假设：

- 路由目录名
- 页面目录名
- 业务模块口头叫法

这三者一定完全一样。

项目里真实存在：

- `operationsManagement` 对应 `src/views/OperationsManage/`
- `money-management` 对应 `src/views/MoneyManagement/`

## 11. 新增独立页面时忘了补注释

这是项目硬规则，不是建议：

- 新增代码必须带有意义的注释
- 尤其是权限兜底、临时 `role` 复用、字段转换、跨菜单跳转这几类地方
