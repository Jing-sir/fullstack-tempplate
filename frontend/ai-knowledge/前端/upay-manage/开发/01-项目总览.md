# 01 项目总览

## 技术栈

从 `package.json` 可以确认当前项目主体技术栈为：

- Vue 3
- TypeScript
- Vite
- Ant Design Vue
- Pinia
- Vue Router
- Vue I18n
- vue-request
- Tailwind CSS
- Sass / SCSS
- Axios

当前仓库使用 `pnpm-lock.yaml`，默认包管理器是 pnpm。

## 目录核心分层

`src` 目录下最值得优先理解的是这些位置：

- `src/main.ts`：应用启动入口，挂载 router、pinia、i18n、全局 mixin、指令和插件
- `src/routes.ts`：把 `asyncRoutes` 与 `constantRoutes` 合并成最终路由表
- `src/setup/router-setup.ts`：路由实例、标题设置、登录鉴权、权限校验
- `src/Main.vue`：后台主布局、侧边菜单、头部、keep-alive、页面加载状态
- `src/routes/`：按业务域拆分的路由模块
- `src/views/`：业务页面
- `src/config-overall/`：配置驱动页面的公共能力
- `src/api/`：业务 API 模块
- `src/servers/`：请求封装、请求头、加密、统一响应解包
- `src/store/`：Pinia store
- `src/mixins/menuMixin.ts`：按钮权限、当前页面按钮菜单计算
- `src/hooks/treeMenu.ts`：菜单树构建、递归查找等全局能力

## 当前项目是混合架构

这个仓库不是单一技术风格，而是长期迭代后的混合型后台项目。

当前主要存在两类页面：

- `config-overall` 配置驱动页面
- 历史遗留的手写页面

真实代码里两类页面都很多，不能假设整个仓库都适合统一重写。

## 入口运行链

项目运行主链可以按下面理解：

1. `src/main.ts` 创建 Vue 应用，并注册 router、pinia、i18n、`menuMixin`、`treeMenu` 等全局能力。
2. `src/routes.ts` 组合 `src/routes/asyncRoutes.ts` 与 `src/routes/constantRoutes.ts`。
3. `src/setup/router-setup.ts` 在路由前置守卫里执行标题设置、语言设置、登录鉴权、权限校验。
4. `src/Main.vue` 负责后台整体布局，并在页面切换时调用 `sideBar` store 拉取菜单与动态路由权限。
5. `src/store/sideBar.ts` 调 `api.sysUserMenuList()` 获取后端菜单，过滤 `asyncRoutes`，生成当前账号可访问的业务路由。

## 路由组成

最终路由由两部分组成：

- `src/routes/constantRoutes.ts`
  - 首页
  - 登录页
  - 错误页
- `src/routes/asyncRoutes.ts`
  - 各业务模块路由
  - 会根据后端返回菜单权限进一步过滤

这意味着：

- 路由文件里写了，不代表最终就一定能进
- 后端菜单数据和 `meta.role` 对不上时，页面仍然会被挡掉

## 页面风格判断

后续开发前，先判断页面属于哪一类：

- 新增独立路由、独立页面、独立菜单：默认走 `config-overall`
- 不新增独立路由，只在现有页面内继续加功能：严格沿用原页面风格

这个判断会直接影响你该改哪些文件、是否能引入新抽象、是否可以重写页面结构。
