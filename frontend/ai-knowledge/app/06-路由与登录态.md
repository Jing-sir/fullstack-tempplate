# 路由与登录态

## 路由中心

- 文件：`lib/lite_router.dart`
- 核心对象：`LiteRouteConfig`
- 功能：
1. 声明路由常量（`static const String ...`）
2. 声明 `getPages`（`List<GetPage>`）
3. 定义默认启动路由（`initialRoute = splash`）

## 登录拦截

- 中间件：`lib/router/check_login_middlewares.dart`
- 逻辑：未登录时重定向到 `LiteRouteConfig.editAccountPage`

登录判定依据：

```dart
String? token = SpUtils.getInstance().get<String>(SpConfig.user_token);
return !(token?.isNotEmpty ?? false);
```

注意：该方法名是 `checkLogin`，但返回 `true` 代表“未登录”。新增逻辑时要避免语义误判。

## 全局同名判定

`LiteRouteConfig.checkLogin()` 与 `CheckLoginMiddleware.checkLogin()` 都使用同一 token 判定方式。涉及登录态变更时应保持一致。

## 新增页面路由步骤

1. 在页面目录创建 `view.dart`（及需要的 `logic.dart`、`state.dart`）。
2. 在 `LiteRouteConfig` 添加路由常量。
3. 在 `getPages` 注册 `GetPage`。
4. 需要登录保护时添加 `middlewares: [CheckLoginMiddleware()]`。
5. 使用 `Get.toNamed(...)` 进行跳转。

## 已知风险

`lite_router.dart` 体量较大，且存在少量重复 `GetPage` 注册。新增路由时应先全局搜索，避免重复 name 导致行为不确定。
