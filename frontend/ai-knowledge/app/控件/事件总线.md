# 事件总线（EventBus）

跨页面、跨模块的通信统一使用全局 `eventBus`，**不要通过 Get.find 层层传回调**。

导入：
```dart
import 'package:flutter_template/config/global_events.dart';
```

---

## 发送事件

```dart
// 触发卡片刷新
eventBus.fire(CardRefresh());

// 触发带数据的事件
eventBus.fire(BannerUpEvent(selectIndex: 0, bean: bannerBean));
```

---

## 监听事件

在 `GetxController.onInit()` 中订阅，在 `onClose()` 中取消：

```dart
class MyLogic extends GetxController {
  StreamSubscription? _sub;

  @override
  void onInit() {
    super.onInit();
    _sub = eventBus.on<CardRefresh>().listen((_) {
      fetchData();
    });
  }

  @override
  void onClose() {
    _sub?.cancel();
    super.onClose();
  }
}
```

## 已有事件一览

| 事件类 | 触发场景 | 典型监听方 |
| --- | --- | --- |
| `CardRefresh` | 卡片数据变更（充值/消费后） | 卡片首页 |
| `CardChange` | 当前选中卡片切换 | 需要展示当前卡信息的页面 |
| `CurrencyChange` | 计价货币切换 | 展示金额的页面 |
| `AssetsChange` | 资产变动 | 资产首页 |
| `RemitChange` | 汇款信息变更 | 汇款相关页面 |
| `LanguageChange` | 语言切换 | 需要重新渲染文案的页面 |
| `LoginEvent` | 登录成功 | 首页、全局状态初始化 |
| `LoginOutEvent` | 登出 | 需要清理状态的页面 |
| `ToHomeEvent` | 跳转回首页 | 导航栏 |
| `ToNotice` | 跳转消息页 | 导航栏 |
| `NoticeRefresh` | 消息列表刷新 | 消息页 |
| `Address` | 地址变更 | 地址相关页面 |
| `BannerUpEvent` | 广告横幅推送 | 广告弹框逻辑 |

---

## 注意事项

- **必须在 `onClose` 取消订阅**，否则页面销毁后仍会收到事件导致崩溃
- 不要在 `StatelessWidget` 里订阅，用 `GetxController` 管理生命周期
- 新增事件类型时，在 `lib/config/global_events.dart` 统一定义，不要散落在业务文件里
