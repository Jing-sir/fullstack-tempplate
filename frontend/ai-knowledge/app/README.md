# upay_app 知识库

这个目录是 `upay_app` 的项目知识库，目标是让新成员可以在短时间内完成：

1. 理解项目技术栈和架构边界。
2. 在本地跑起 QA/Prod 等不同 flavor。
3. 按现有模式新增页面、接口和路由。
4. 排查常见网络、登录态、打包问题。

## 维护规则

1. 代码结构发生变化时，同步更新对应文档。
2. 新增基础能力时，优先补充到对应章节，而不是只写在 PR 描述里。
3. 命令、路径、配置项必须以仓库当前实现为准（不要写"猜测值"）。
4. 对历史问题和坑点，优先记录"触发条件 + 复现 + 处理步骤"。
