# 主题系统：深色模式 / 浅色模式

## 概述

upay_app 使用 GetX 的 `ThemeMode` 机制管理深色/浅色主题，支持三种模式：**跟随系统 / 浅色 / 深色**。用户选择持久化到 SharedPreferences，图片资源随主题自动切换。

---

## 文件结构

```
lib/theme/
  index.dart              # CustomTheme 入口，暴露 light / dark 两个 ThemeData
  LightThemeData.dart     # 浅色主题完整配置
  DarkThemeData.dart      # 深色主题完整配置
  ThemeImage.dart         # 主题图片路由（自动按 isDarkMode 返回对应资源路径）
  styles.dart             # PageStyle 颜色常量 + 语义色（theme-aware）

lib/lite_page/mine/lite_switch_theme/
  logic.dart              # 切换逻辑
  state.dart              # 当前选中的 themeType（0/1/2）
  view.dart               # 设置页 UI

lib/utils/utils.dart      # getSystemThemeMode() / getSystemThemeTitle()
lib/config/sp_config.dart # SpConfig.themeType = "themeType"（持久化 key）
lib/global_app/logic.dart # 初始化时读取 isDark 状态
lib/global_app/view.dart  # 平台亮度变化监听 + GetMaterialApp themeMode 配置
```

## 主题值说明

| themeType | ThemeMode | 含义 |
|-----------|-----------|------|
| 0 | `ThemeMode.system` | 跟随系统（默认） |
| 1 | `ThemeMode.light`  | 强制浅色 |
| 2 | `ThemeMode.dark`   | 强制深色 |

持久化 key：`SpConfig.themeType`（值为 int，存于 SharedPreferences）

---

## 主题切换入口

**[lib/lite_page/mine/lite_switch_theme/logic.dart](lib/lite_page/mine/lite_switch_theme/logic.dart)**

```dart
switchTheme(int themeType) {
  Get.changeThemeMode(ThemeMode.system / light / dark);
  SpUtils.getInstance().setData(SpConfig.themeType, themeType);
  state.themeType.value = themeType;
  Future.delayed(Duration(milliseconds: 350), () => Get.forceAppUpdate());
}
```

切换流程：
1. `Get.changeThemeMode(...)` — 立即切换 GetX 主题
2. `SpUtils.setData(SpConfig.themeType, ...)` — 持久化
3. `Get.forceAppUpdate()` — 延迟 350ms 强制刷新，确保图片/样式全量更新

## 主题感知图片：ThemeImage

**[lib/theme/ThemeImage.dart](lib/theme/ThemeImage.dart)**

所有需要随主题切换的图片都在此类中统一定义，通过 `Get.isDarkMode` 返回对应资源路径：

```dart
// 用法示例
Image.asset(ThemeImage.no_data)         // 自动返回 no_data.png 或 no_data_dark.png
Image.asset(ThemeImage.bg_tips_card)
```

**命名规范**：深色资源文件名加 `_dark` 后缀（如 `no_data_dark.png`）。

---

## 主题感知颜色：PageStyle

**[lib/theme/styles.dart](lib/theme/styles.dart)**

语义色通过 `Get.isDarkMode` 动态返回，例如：

```dart
static Color get outlineColor => Get.isDarkMode ? c_121212 : c_F6F7F8;
static Color get headlineColor => Get.isDarkMode ? c_FFFFFF : c_111A34;
```

新增颜色优先使用语义色 getter，不要硬编码 hex 值。

## 平台亮度变化处理

**[lib/global_app/view.dart:60](lib/global_app/view.dart)**

```dart
void didChangePlatformBrightness() async {
  Future.delayed(Duration(milliseconds: 250), () {
    if (appLogic.state.isDark.value != Get.isPlatformDarkMode) {
      appLogic.state.isDark.value = Get.isPlatformDarkMode;
      Get.forceAppUpdate();
    }
  });
}
```

仅当 themeType == 0（跟随系统）时，系统亮度变化才会实际影响 UI（GetX 内部处理）。

## 新增主题感知图片的标准流程

1. 准备两张图片：`xxx.png`（浅色）和 `xxx_dark.png`（深色）
2. 放入 `assets/images/` 目录，并在 `pubspec.yaml` 中声明
3. 在 `ThemeImage.dart` 新增 getter：
   ```dart
   static String get xxx => checkUri(Get.isDarkMode ? 'xxx_dark' : 'xxx');
   ```
4. 使用时：`Image.asset(ThemeImage.xxx)`

---

## 新增主题感知颜色的标准流程

1. 在 `styles.dart` 的颜色常量区添加 hex 常量（如已存在则跳过）
2. 添加语义色 getter：
   ```dart
   static Color get xxxColor => Get.isDarkMode ? c_暗色值 : c_亮色值;
   ```
3. 页面中使用 `PageStyle.xxxColor` 而不是直接写 hex

---

## 注意事项

- **不要**在 Widget build 时直接写 `Get.isDarkMode ? Color(0xFF...) : Color(0xFF...)`，统一收敛到 `PageStyle` 或 `ThemeImage`
- **不要**直接调用 `Get.changeThemeMode` 而跳过 `SpUtils` 持久化，否则重启后丢失用户设置
- `Get.forceAppUpdate()` 会重建整个 Widget 树，非必要不要在高频场景调用
- SVG 资源使用 `checkSvgUri`，PNG 使用 `checkUri`（见 ThemeImage 底部工具方法）
