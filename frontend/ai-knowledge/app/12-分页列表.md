# 分页列表：下拉刷新 + 上滑加载更多

## 概述

upay_app 有两个封装好的分页组件，均通过 `onRefresh` / `onLoadMore` 回调驱动，内部处理加载状态显示。**不使用第三方 EasyRefresh / pull_to_refresh 包**。

---

## 核心组件

### CustomScrollBar

**路径**：[lib/widgets/custom_scroll_bar.dart](lib/widgets/custom_scroll_bar.dart)

最常用版本。内部使用 `RefreshIndicator` + `ListView` + `ScrollController`。

```dart
CustomScrollBar(
  onRefresh: logic.onRefresh,         // Future<void>，下拉刷新
  onLoadMore: logic.onLoadMore,       // Future<bool>，上拉加载（返回 true=有更多）
  pageSize: CommonConfig.pageSize,    // 每页条数，决定是否显示底部加载提示
  padding: EdgeInsets.all(16.w),
  children: state.items.map(_buildItem).toList(),
)
```

- `onLoadMore` 返回 `true` → 继续显示"正在加载"；返回 `false` → 显示"已全部显示"
- `children.length < pageSize` 时不显示底部加载提示（认为数据量不足一页，不再分页）

### CustomScrollBar2

**路径**：[lib/widgets/custom_scroll_bar2.dart](lib/widgets/custom_scroll_bar2.dart)

与 CustomScrollBar 逻辑相同，额外支持 `ScrollController` 外部传入，适合需要监听滚动偏移量的场景。

## 标准 Logic 模式

```dart
class XxxLogic extends GetxController {
  final XxxState state = XxxState();
  static const _pageSize = CommonConfig.pageSize; // 20

  // 骨架屏首次加载 (ShimmerLoadingView 调用)
  Future<void> skeletonRefresh() async {
    final data = await Api.fetchList(pageNo: 1, pageSize: _pageSize);
    state.items.value = data;
    state.pageNo = 1;
    state.hasMore.value = data.length >= _pageSize;
  }

  // 下拉刷新（onRefresh 回调）
  Future<void> onRefresh() async {
    final data = await Api.fetchList(pageNo: 1, pageSize: _pageSize);
    state.items.value = data;
    state.pageNo = 1;
    state.hasMore.value = data.length >= _pageSize;
  }

  // 上拉加载更多（onLoadMore 回调，返回 bool）
  Future<bool> onLoadMore() async {
    final next = state.pageNo + 1;
    final data = await Api.fetchList(pageNo: next, pageSize: _pageSize);
    state.items.addAll(data);
    state.pageNo = next;
    final hasMore = data.length >= _pageSize;
    state.hasMore.value = hasMore;
    return hasMore;
  }
}
```

## 标准 State 模式

```dart
class XxxState {
  var items = <XxxItem>[].obs;
  var pageNo = 1;         // int，不需要 .obs（CustomScrollBar 自己管理加载状态）
  var hasMore = true.obs; // 可选，用于 NoDataView 的 hasData 判断
}
```

**注意**：`isLoadingMore` 不需要放到 State — 由 `CustomScrollBar` 内部的 `_LoadingMoreState` 管理。

## 配合 NoDataView（空数据态）

当列表可能为空时，在 `CustomScrollBar` 外层包一层 `NoDataView`：

```dart
Obx(() => NoDataView(
  hasData: state.items.isNotEmpty,
  loading: logic.onRefresh,  // 也可以是 skeletonRefresh
  child: CustomScrollBar(
    onRefresh: logic.onRefresh,
    onLoadMore: logic.onLoadMore,
    pageSize: CommonConfig.pageSize,
    children: state.items.map(_buildItem).toList(),
  ),
))
```

- `loading` 函数在 `initState` 时自动执行一次，展示内部 loading 动画
- 数据为空时渲染"没有可显示的记录"（可通过 `noTitle` 自定义）

## 配合 Sliver 吸顶（NestedScrollView 模式）

当页面既需要吸顶 header 又需要分页列表时，使用 `NestedScrollView`：

```dart
NestedScrollView(
  headerSliverBuilder: (context, innerBoxIsScrolled) => [
    // 随内容滚出的区域（卡片摘要等）
    SliverToBoxAdapter(child: _summaryCard()),
    // 吸顶标题
    SliverPersistentHeader(
      pinned: true,
      delegate: _StickyHeaderDelegate(height: 44.w, child: _stickyTitle()),
    ),
  ],
  body: Obx(() => CustomScrollBar(
    onRefresh: logic.onRefresh,
    onLoadMore: logic.onLoadMore,
    pageSize: CommonConfig.pageSize,
    children: state.items.map(_buildItem).toList(),
  )),
)
```

吸顶 Header Delegate：

```dart
class _StickyHeaderDelegate extends SliverPersistentHeaderDelegate {
  final Widget child;
  final double height;
  _StickyHeaderDelegate({required this.child, required this.height});

  @override
  Widget build(context, shrinkOffset, overlapsContent) => child;
  @override double get maxExtent => height;
  @override double get minExtent => height;
  @override bool shouldRebuild(_StickyHeaderDelegate old) =>
      old.child != child || old.height != height;
}
```

## pageSize

全局常量在 `CommonConfig.pageSize = 20`，统一使用，不要硬编码。

---

## 注意事项

- `CustomScrollBar` 内部自己持有 `ScrollController`，不需要外部传入（除非需要监听滚动偏移量，此时用 `CustomScrollBar2`）
- `onLoadMore` 必须返回 `Future<bool>`，**不能** 是 `Future<void>`
- `children.length < pageSize` 时底部加载提示自动隐藏，无需额外判断
- `NestedScrollView` + `CustomScrollBar` 组合中，`CustomScrollBar` 需要设置 `physics: const ClampingScrollPhysics()` 以避免双重弹性滚动冲突（视实际效果调整）
