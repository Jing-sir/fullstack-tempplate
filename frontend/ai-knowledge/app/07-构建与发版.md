# 构建与发版

## Android 构建命令

来自根 `README.md` 的现有命令如下。

QA APK：

```bash
flutter build apk --flavor qa -t lib/main_qa.dart --obfuscate --split-debug-info=./android_debug_info
```

Prod APK：

```bash
flutter build apk --flavor prod -t lib/main_prod.dart --obfuscate --split-debug-info=./android_debug_info
```

Google AAB：

```bash
flutter build appbundle --flavor google -t lib/main_google.dart --obfuscate --split-debug-info=./android_debug_info
```

Huawei APK：

```bash
flutter build apk --flavor huawei -t lib/main_huawei.dart --obfuscate --split-debug-info=./android_debug_info
```

## Android Flavor 配置

`android/app/build.gradle` 当前定义：

- `prod`：`com.upay.best`
- `google`：`com.upay.best`
- `huawei`：`com.upay.best`
- `qa`：`com.upay.best.qa`

同时开启了输出文件命名模板：`UPB-v{versionName}({versionCode})-{timestamp}`。

## iOS 打包备注（历史问题）

README 中记录过 iOS 18 上传 Bitcode 问题（`ZendeskSDKLogger.framework`）：

1. 使用 `otool` 检查是否包含 `__LLVM` 段。
2. 需要时使用 `xcrun bitcode_strip ... -r -o ...` 去除 bitcode。
3. 重新打包上传。

## 发版前检查建议

1. flavor 与入口文件一致（避免 `prod` flavor 跑 `qa` main）。
2. API Host 与 Firebase 配置正确。
3. `android_debug_info` 与产物一并归档。
4. 至少完成 `flutter analyze` 与核心路径冒烟测试。
