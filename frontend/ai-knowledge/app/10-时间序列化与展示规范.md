# 时间序列化与展示规范

## 核心工具类

`lib/utils/date_util.dart` — 项目唯一时间处理工具，**所有时间格式化必须使用此类，禁止直接使用 `DateFormat`（intl）或手写格式化**。

---

## API 时间字段的数据格式

项目中时间字段常见两种形式：

| 形式 | 示例 | 说明 |
|---|---|---|
| 毫秒时间戳（int） | `1712345678000` | 部分字段如 `transactionTime`、`ts` 直接为 int |
| 毫秒时间戳字符串 | `"1712345678000"` | 部分字段如 `createTime`、`depositTime` 是字符串形式的毫秒戳 |
| ISO 字符串 | `"2024-04-05 14:30:00"` | 少数接口返回标准格式字符串 |

**反序列化时不要假设类型**，用安全转换：

```dart
// 字符串毫秒戳 → 格式化时间
DateUtil.getDateStrByMs(int.tryParse(item.createTime ?? '0') ?? 0) ?? '--'

// int 毫秒戳 → 格式化时间  
DateUtil.getDateStrByMs(item.transactionTime ?? 0) ?? '--'

// ISO 字符串 → DateTime
DateUtil.getDateTime(dateStr)
```

## 常用格式化方法

### getDateStrByMs（最常用）

```dart
// 默认 yyyy-MM-dd HH:mm:ss
DateUtil.getDateStrByMs(milliseconds) 

// 指定格式
DateUtil.getDateStrByMs(milliseconds, format: DateFormat.YEAR_MONTH_DAY)
// → "2024-04-05"

DateUtil.getDateStrByMs(milliseconds, format: DateFormat.MONTH_DAY_HOUR_MINUTE)
// → "04-05 14:30"

DateUtil.getDateStrByMs(milliseconds, format: DateFormat.HOUR_MINUTE)
// → "14:30"
```

### getTimeOld（账单列表日期分组标题）

```dart
DateUtil.getTimeOld(DateUtil.getDateTimeByMs(milliseconds))
// 今天 → "今天"
// 昨天 → "昨天"  
// 本年其他日期 → "04-05"
// 跨年 → "2023-04-05"
```

### uponTime（相对时间，如通知消息）

```dart
DateUtil.uponTime(milliseconds)
// → "3分钟前" / "2小时前" / "昨天" / "3天前"
```

## DateFormat 枚举

```dart
DateFormat.DEFAULT                   // yyyy-MM-dd HH:mm:ss.SSS
DateFormat.NORMAL                    // yyyy-MM-dd HH:mm:ss  ← 默认值
DateFormat.YEAR_MONTH_DAY_HOUR_MINUTE // yyyy-MM-dd HH:mm
DateFormat.YEAR_MONTH_DAY            // yyyy-MM-dd
DateFormat.YEAR_MONTH                // yyyy-MM
DateFormat.MONTH_DAY                 // MM-dd
DateFormat.MONTH_DAY_HOUR_MINUTE     // MM-dd HH:mm
DateFormat.HOUR_MINUTE_SECOND        // HH:mm:ss
DateFormat.HOUR_MINUTE               // HH:mm

// 中文格式（ZH_ 前缀）
DateFormat.ZH_YEAR_MONTH_DAY         // yyyy年MM月dd日
DateFormat.ZH_YEAR_MONTH_DAY_HOUR_MINUTE // yyyy年MM月dd日 HH时mm分
// ...
```

## 场景对应表

| 使用场景 | 推荐方法 | 格式 |
|---|---|---|
| 订单/账单列表时间行 | `getTimeOld` | 今天/昨天/MM-dd/yyyy-MM-dd |
| 订单/账单条目时间 | `getDateStrByMs` | `HOUR_MINUTE` 或 `MONTH_DAY_HOUR_MINUTE` |
| 详情页完整时间 | `getDateStrByMs` | `NORMAL`（默认） |
| 通知消息时间 | `getDateStrByMs` | `MONTH_DAY_HOUR_MINUTE` |
| 相对时间描述 | `uponTime` | "3分钟前" |

---

## 安全规则

1. **永远加 `?? '--'`**：`getDateStrByMs` 可能返回 `null`（毫秒为 0 时返回 `"--"`，但其他格式可能返回 null），展示时兜底用 `'--'`。
2. **字符串字段先 `int.tryParse`**：`createTime` 等字段为字符串，必须用 `int.tryParse(str ?? '0') ?? 0` 转换后传入。
3. **不要手写 `substring` 截时间**：项目内存在历史代码用字符串截取格式化时间，新代码禁止，统一走 `DateUtil`。
4. **不要引入 `intl` 包**：项目已有 `DateUtil` 覆盖所有常见格式，无需 `DateFormat` from intl。
