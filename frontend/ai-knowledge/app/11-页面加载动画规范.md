# 页面加载动画规范

## 结论：禁止使用 CircularProgressIndicator 作为全页加载

本项目统一使用 **骨架屏 + Shimmer 动效** 作为页面初始加载态，**不得**使用 Flutter 原生的 `CircularProgressIndicator`（转圈）作为全页加载占位。

---

## 项目标准方案

### 1. 核心组件

`lib/widgets/ShimmerLoadingView.dart` — **骨架屏加载容器**

```dart
ShimmerLoadingView(
  shimmerChild: skeletonXxx(),  // 骨架屏占位 Widget
  loading: logic.skeletonRefresh, // 触发数据加载的 Future Function()
  child: Obx(() { /* 数据加载完成后的真实内容 */ }),
)
```

- `shimmerChild`：与真实页面结构相似的灰色占位块（来自 `lib/widgets/skeleton/` 目录）
- `loading`：一个返回 `Future` 的函数，`ShimmerLoadingView` 在 `initState` 时自动调用，完成后切换为 `child`
- Shimmer 颜色使用语义色：`PageStyle.skeletonColor` / `PageStyle.skeletonHighColor`（自动适配深色/浅色主题）

### 2. 骨架屏文件

所有骨架屏占位 Widget 存放于 `lib/widgets/skeleton/`，命名规则：`skeleton_<业务模块>.dart`，导出顶层函数（非 class），例如：

```dart
Widget skeletonMyCard() { ... }
Widget skeletonCardHeader() { ... }
```

颜色固定用 `const Color(0xFFEBEEF2)`（浅灰色块），宽高比例模仿真实内容，使用 `BorderRadius.circular(2.w)` 或 `8.w`。

### 3. Logic 层对应方法

在 Logic 中暴露一个 `skeletonRefresh()` 方法，内部并发调用所有首屏数据接口：

```dart
Future skeletonRefresh() async {
  await Future.wait([
    fetchDataA(),
    fetchDataB(),
  ]);
}
```

`ShimmerLoadingView` 会持有这个 Future，自动管理 loading → 完成 的状态切换。

## 错误示范（不要这样写）

```dart
// ❌ 错误：不要用 CircularProgressIndicator 做全页加载
if (state.isLoading.value) {
  return const Center(child: CircularProgressIndicator());
}
```

---

## 例外场景

| 场景 | 允许的组件 |
|---|---|
| 按钮内嵌 loading（如保存中） | `CircularProgressIndicator(strokeWidth: 2)` ✅ |
| 列表底部分页加载更多 | `CircularProgressIndicator(strokeWidth: 2)` ✅ |
| 全页初始加载 | `ShimmerLoadingView` + `skeleton_xxx` ✅ |
| 全局操作（如提交表单） | `EasyLoading.show()` ✅ |

---

## 新页面开发步骤

1. 在 `lib/widgets/skeleton/` 新增 `skeleton_<模块>.dart`，绘制与真实页面相似的占位布局
2. Logic 中新增 `skeletonRefresh()` 方法
3. View 中用 `ShimmerLoadingView` 包裹内容区域，传入 `shimmerChild` 和 `loading`
4. **不需要**在 State 里维护 `isLoading` 变量（由 `ShimmerLoadingView` 内部管理）
