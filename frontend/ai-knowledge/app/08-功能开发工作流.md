# 功能开发工作流

## 目标

给出一个和当前项目结构一致的“新增功能最小闭环”流程，避免出现：

- 页面写好了但未注册路由
- 请求接了但错误码未处理
- 文案缺失导致多语言回退
- 登录保护遗漏

## 标准流程

1. 需求拆分  
确认是新页面、弹窗流程、还是仅接口变更。

2. 页面骨架  
在目标域目录下创建 `view.dart`，按需补 `logic.dart`、`state.dart`。

3. 路由接入  
更新 `lib/lite_router.dart`：
- 新增路由常量
- `getPages` 注册
- 涉及登录能力时加 `CheckLoginMiddleware`

4. 数据接入  
更新 `lib/apis/apis.dart` 与对应 `lib/apis/*.dart`，在页面逻辑层调用。

5. 国际化  
补齐 `lib/language/translation_*.dart` 需要的 key。

6. 主题与资源  
复用 `lib/theme/` 与 `assets/` 规范，避免硬编码颜色与尺寸。

7. 自测  
至少覆盖：
- 未登录与已登录路径
- 弱网/超时/服务端报错提示
- 深链或回退路径是否正常

## 提交前检查清单

1. 路由名唯一，无重复注册。
2. API 常量命名与业务文件归类一致。
3. 新增存储 key 已纳入清理策略（若与登录态强相关）。
4. 关键文档已更新（本知识库对应章节）。
