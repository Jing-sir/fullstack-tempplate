# 测试分层规范

## 1. 测试层级定义

### L1: 单元测试
- **范围**: 函数/类/模块级别
- **工具**: [Jest/Vitest/JUnit/pytest]
- **覆盖率目标**: ≥80%
- **执行时机**: 每次代码提交前
- **责任人**: 开发工程师

### L2: 集成测试
- **范围**: 模块间交互、数据库、外部服务
- **工具**: [Testcontainers/MockServer]
- **执行时机**: PR合并前
- **责任人**: 开发工程师 + QA

### L3: 接口测试
- **范围**: API 契约、请求响应、鉴权
- **工具**: [Postman/Newman/REST Assured]
- **执行时机**: 每日定时 / 部署前
- **责任人**: QA 工程师

### L4: UI 测试
- **范围**: 关键用户流程、跨浏览器
- **工具**: [Playwright/Cypress]
- **执行时机**: 回归测试阶段
- **责任人**: QA 自动化工程师

### L5: 端到端测试
- **范围**: 完整业务流程
- **工具**: [自定义脚本/商业工具]
- **执行时机**: 预发布环境 / 上线前
- **责任人**: QA 团队
