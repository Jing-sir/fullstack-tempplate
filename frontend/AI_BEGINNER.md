# AI 小白派单入口

这份文件给完全不懂前端的人用。你不需要知道 `npm`、`pnpm`、Vue、组件、路由、接口封装是什么意思。

你只负责说清楚业务。技术路线、代码实现、验证命令，由 AI 按 `AGENTS.md` 和 `AI_TASK_GUIDE.md` 负责。

项目里已经保留了标准代码模板：`ai-templates/`。以后业务页面删掉后，AI 也必须按这里的模板继续写；新增大菜单、小页面、详情/编辑隐藏页时，必须参考 `ai-templates/routes/README.md`。

## 1. 你只要做三件事

1. 复制下面的“万能派单提示词”。
2. 把你知道的业务信息填进去；不知道就写“待确认”。
3. 看 AI 最后有没有给出“验证结果”和“需要人工确认”。

## 2. 万能派单提示词

```text
请先阅读 AGENTS.md、AI_BEGINNER.md、AI_TASK_GUIDE.md。

我是非前端人员，请不要要求我解释技术实现。
你必须按项目现有架构实现，不能自己发明写法。
你不能直接开始写代码，请先输出“开工前确认”。

我要做的功能：
- 

页面/模块名称：
- 

页面上需要看到：
- 搜索条件：
- 表格列：
- 哪些 UID / 编号 / 单号需要复制：
- 按钮：
- 弹窗/抽屉：
- 成功提示：

接口信息：
- 我知道的接口：
- 我不知道的接口：

状态含义：
- 我知道的状态：
- 我不知道的状态：

权限信息：
- 我知道的权限：
- 我不知道的权限：

验收标准：
- 

请你先告诉我：
1. 你判断这是什么任务类型？
2. 你会参考哪些现有文件？
3. 你计划改哪些文件？
4. 哪些信息你能从项目里查，哪些必须我确认？
5. 是否需要先走 OpenSpec / 提案流程？
6. 你最后会运行哪些验证命令？
```

## 3. 不知道怎么填时

按这个简单版本填也可以：

```text
我要做的功能：
- 在 XXX 页面增加 XXX 功能

页面上需要看到：
- 搜索条件：有哪些就写哪些
- 表格列：有哪些就写哪些
- 需要复制的字段：例如用户UID、订单编号；没有就写无
- 按钮：有哪些就写哪些
- 弹窗：有没有新增/编辑/详情/审核/冻结等弹窗

我知道的接口：
- 有就写接口地址；没有就写待确认

状态含义：
- 例如 1=正常，2=冻结；不知道就写待确认

验收标准：
- 我希望页面最终表现成什么样
```

## 4. 你不用回答这些技术问题

如果 AI 问下面这些问题，让它自己根据项目规则判断：

- 要不要用 `TableSearchWrap`？
- API 文件应该放哪里？
- 路由应该写在哪个文件？
- 弹窗应该放 `components` 还是 `modal`？
- 要不要运行 `pnpm run typecheck`？
- 要不要运行 `pnpm run ai:check`？

这些是 AI 的责任，不是你的责任。

## 5. 你需要确认的业务问题

如果 AI 问下面这些问题，你需要尽量确认：

- 状态值是什么意思，例如 `0/1/2` 分别代表什么。
- 按钮有没有权限 key，例如新增、编辑、删除、导出。
- 接口地址、请求方式、参数名、返回字段是否确定。
- 成功后是刷新列表、关闭弹窗，还是跳转页面。
- 字段展示的是编号、名称、金额、时间，还是状态文案。

不知道就直接说“我不知道，请你先从项目或旧项目里查；查不到再列为风险”。

## 6. AI 最后必须交付这些

AI 最后回复必须包含：

```text
完成内容：
修改文件：
接口和字段确认：
仍需人工确认：
验证结果：
业务验收点：
风险说明：
```

如果没有“验证结果”，就不能算完成。

## 7. 验证结果怎么看

你不需要懂命令，只看结论：

- `pnpm run typecheck：通过` 说明类型检查通过。
- `pnpm exec eslint <改动文件>：通过` 说明改动文件代码规范检查通过。
- `pnpm run ai:check：通过` 说明没有明显违反项目架构路线。

如果显示“未运行”或“未通过”，让 AI 说明原因并继续处理。
